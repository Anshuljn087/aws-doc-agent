# Amazon API Gateway basics

Amazon API Gateway is a fully managed service for creating, publishing, maintaining, and securing APIs at any scale. It can expose HTTP, REST, or WebSocket APIs and route requests to backend services such as Lambda, EC2, or other AWS resources. API Gateway is often used to build public or private application interfaces for web and mobile clients.

API Gateway provides features such as request validation, throttling, authentication, and request/response transformation. Developers can define API stages, such as development, staging, and production, to manage deployment versions. This makes it easier to roll out changes in a controlled way while keeping the API stable for consumers.

The service supports both REST APIs and HTTP APIs. REST APIs offer broader feature support for enterprise use cases, while HTTP APIs are generally simpler and faster to configure for common web workloads. Both options can integrate with AWS Lambda for serverless application backends.

API Gateway is a key piece of many AWS application architectures because it centralizes access control and request handling. It helps teams expose business logic in a consistent and scalable interface while keeping backend services decoupled from clients.
