# Amazon CloudWatch basics

Amazon CloudWatch is AWS's monitoring and observability service for metrics, logs, alarms, and events. It helps teams understand the health and performance of applications and infrastructure running in AWS. CloudWatch collects operational data from services such as EC2, Lambda, API Gateway, and RDS.

Metrics are the primary data type in CloudWatch and are used for dashboards, alarms, and automated responses. Logs can be collected from applications and services and searched to troubleshoot issues. CloudWatch alarms can notify users or trigger actions when metrics cross defined thresholds, which makes it useful for incident response.

CloudWatch also supports events that deliver near real-time notifications about changes in AWS resources. These events can be routed to targets such as Lambda, SNS, or Step Functions. This integration enables automated workflows for maintenance, scaling, and operational responses.

For developers and operators, CloudWatch is an essential tool for visibility. It provides the signals needed to detect failures early, diagnose issues, and improve the reliability of AWS-based systems.
